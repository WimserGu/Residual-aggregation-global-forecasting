# Residual Aggregation under Partial Market Coverage

This repository accompanies the manuscript:

> **When and Why Does Residual Aggregation Improve Global Forecasting under Partial Market Coverage?**

This repository is designed to support two forms of reproducibility:

1. **Methodological reproducibility.** The code implements the full experimental framework: Top-K coverage construction, residual series construction, DirectK and DirectK+Residual global forecasting, rolling-origin evaluation, placebo auxiliary-series experiments, representation ablation, bootstrap summaries, and figure/table generation.
2. **Exact numerical reproduction.** Exact replication of the manuscript's numerical results requires the historical Norwegian passenger-vehicle registration data used in the study. These data are not redistributed in this repository because redistribution rights may be restricted by the original provider.

The manuscript outputs are archived under `results/manuscript/` and `figures/manuscript/`. New runs write to `results/reproduced/` and `figures/reproduced/`, so rerunning the workflow will not overwrite the manuscript archive.

## Data Requirements

The final experiments require a processed long-format CSV:

```text
data/processed/monthly_market_series.csv
```

Required columns:

| Column | Description |
| :--- | :--- |
| `date` | Monthly timestamp, formatted as the first day of the month. |
| `brand` | One of the 14 observed manuscript brands. |
| `sales` | Monthly passenger-vehicle registrations for the observed brand. |
| `market_size` | Official total passenger-vehicle registrations for the same month. |

The balanced panel should contain the 14 observed brands for each month from January 2018 to March 2026:

```text
Audi, BMW, Ford, Hyundai, Kia, Mazda, Mercedes-Benz, Nissan,
Peugeot, Skoda, Tesla, Toyota, Volkswagen, Volvo
```

The residual category is constructed inside the experiment code as:

```text
Residual_t = market_size_t - sum(selected_brand_sales_t)
```

## Raw Data Source and Access Procedure

The manuscript uses monthly Norwegian passenger-vehicle registration data from **OFV Registreringsstatistikk** and monthly official total market size for the same period.

The original development files were:

```text
data/raw/Norway_monthly_car_sale.xlsx
data/raw/All_indicators.xlsx
```

Expected sheet names and columns:

| File | Sheet | Required columns |
| :--- | :--- | :--- |
| `Norway_monthly_car_sale.xlsx` | `Sheet2` | `Brand`, `Sales`, `date` |
| `All_indicators.xlsx` | `Ark1` | `Month`, `market_size` |

Only `market_size` is required from `All_indicators.xlsx` for the final residual-aggregation experiments. Other columns in the development file are not used by the final public workflow.

To prepare the processed CSV from licensed raw files:

```bash
python scripts/prepare_processed_data.py
```

The historical dataset was assembled from repeated downloads of OFV registration statistics during the course of this project rather than obtained through a single download. Accordingly, no single access date applies to the complete dataset. Users seeking exact numerical reproduction should document the access dates and data extracts used for their own licensed copy of the data.

## Synthetic Smoke Test

If licensed data are unavailable, a schema-compatible synthetic sample can be generated:

```bash
python scripts/prepare_processed_data.py --synthetic-sample --output data/processed/monthly_market_series.csv
```

This checks that the workflow executes, but it does **not** reproduce the manuscript's numerical results.

## Automated Checks

The repository includes lightweight tests for data-schema validation, leakage-safe lag construction, deterministic Top-K brand selection, and metric edge cases. To run them locally:

```bash
pip install -e ".[test]"
pytest
```

The GitHub Actions smoke-test workflow runs these tests and verifies that a schema-compatible synthetic sample can be generated. The workflow is intentionally lightweight and does not rerun the full manuscript experiments.

## Reproducing Outputs

After preparing `data/processed/monthly_market_series.csv`, run:

```bash
python scripts/run_all_experiments.py
```

This writes new reproducibility outputs to:

```text
results/reproduced/
figures/reproduced/
```

The archived manuscript outputs remain unchanged in:

```text
results/manuscript/
figures/manuscript/
```

For a faster check that skips the 20-seed placebo experiments:

```bash
python scripts/run_all_experiments.py --skip-placebos
```

## Individual Scripts

```bash
python scripts/run_coverage_ladder.py
python scripts/run_representation_ablation.py
python scripts/run_placebos.py
python scripts/make_tables.py
python scripts/make_figures.py
```

By default, individual scripts read from `data/processed/monthly_market_series.csv` and write to `results/reproduced/` or `figures/reproduced/`.

Files under `results/manuscript/` and `figures/manuscript/` are archived manuscript outputs. The archived manuscript figures are provided in PNG format only, because these are the final publication-quality versions used in the submitted manuscript. Files under `results/reproduced/` and `figures/reproduced/` are generated by local reruns and are intentionally separated from the archived outputs.

## Placebo Naming

The repository uses the following placebo labels consistently:

| Label | Meaning |
| :--- | :--- |
| `ShuffledResidual` | Authentic residual values randomly permuted within the training window. |
| `SyntheticResidual` | Synthetic AR(1)-style series matched to basic residual properties. |
| `DuplicateAuxiliary` | A duplicated observed series selected deterministically within each fold. |
| `ObservedBrandAggregate` | Random aggregate of already observed brands, scaled to the residual's training-window mean. |

## Coverage Definition

The manuscript reports observed coverage using the average share of the selected Top-K brands during the initial 48-month training window. This is the coverage definition used by the table and plotting scripts.

The metadata file also contains a full-sample descriptive coverage field, but manuscript tables and figures use:

```text
initial_training_coverage
```
